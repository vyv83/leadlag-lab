Standalone menu prototype for pipeline-first UX

Prototype Direction

Menu shaped by the real research pipeline.

This template uses project-true terminology, ownership, and actions: collector leads into recordings, recordings own analyses, strategies own backtests and paper runs. It is intentionally standalone and backend-free so we can iterate on structure and control density first.

Prototype Rules

No legacy session/collection language in UX. Collector is shown as runtime inside the app, not as a separate systemd service. Quality and Explorer stay as context actions for analyses, not as top-level ownership branches.
